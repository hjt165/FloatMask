# FloatMask - 智能字幕遮挡工具

## 项目概述
FloatMask是一款基于Android平台的悬浮窗工具应用，旨在帮助外语学习者通过遮挡字幕来提升听力能力。用户可以自由拖动和调整悬浮框大小，将其覆盖在视频字幕区域，实现无字幕听力训练。

---

## 功能设计

### 核心功能（必须实现）
| 功能模块 | 具体功能 | 说明 |
|---------|---------|------|
| 悬浮框显示与管理 | 权限检测 | App启动时检测悬浮窗权限，无权限则引导开启 |
| | 悬浮框创建 | 在屏幕上显示半透明带边框的悬浮方框 |
| | 初始状态 | 初始尺寸200×120像素，位置为屏幕中央偏右 |
| | 退出清理 | App关闭后悬浮框完全消失，不残留后台进程 |
| 交互功能 | 拖动移动 | 支持单指拖动改变位置，拖动过程实时更新 |
| | 尺寸调整 | 支持调整悬浮框大小 |
| | 边界限制 | 悬浮框不能完全拖出屏幕（至少保留20%在可视区域内） |
| | 尺寸限制 | 最小80×80像素，最大屏幕宽度30%×屏幕高度30% |
| 视频遮挡功能 | 覆盖能力 | 悬浮框显示在任何第三方视频App上方 |
| | 遮挡效果 | 有效遮挡字幕区域 |
| | 点击穿透 | 点击悬浮框区域不影响下方视频播放控制 |

### 拓展功能（选做加分项）
| 功能模块 | 具体功能 | 说明 |
|---------|---------|------|
| 增强交互 | 颜色切换 | 双击切换半透明灰色/半透明黑色/纯黑色不透明 |
| | 快捷菜单 | 长按弹出菜单，支持调节透明度、关闭悬浮框 |
| | 记忆功能 | 下次启动时自动恢复上次的位置和大小 |
| | 最小化模式 | 点击箭头按钮缩成小圆点，再次点击恢复 |

### 异常处理
| 异常类型 | 处理方案 |
|---------|---------|
| 权限异常 | 用户未授权时给出明确提示并引导跳转设置页面 |
| 服务保活 | 防止系统因内存不足自动关闭悬浮窗服务 |
| 兼容处理 | 针对不同Android版本和厂商（华为、小米、OPPO等）的悬浮窗差异进行处理 |

## 页面设计

### 1. 启动页面（Splash Screen）
- **作用**：应用启动时显示，用于加载资源和检测权限
- **显示内容**：应用Logo、应用名称、加载进度条
- **跳转逻辑**：加载完成后自动跳转到权限检测页面

### 2. 权限检测页面（Permission Page）
- **作用**：检测并引导用户开启悬浮窗权限
- **显示内容**：
  - 权限状态图标（已授权/未授权）
  - 权限说明文字
  - "开启权限"按钮（未授权时显示）
  - "进入应用"按钮（已授权时显示）
- **交互逻辑**：
  - 点击"开启权限"跳转到系统悬浮窗权限设置页面
  - 权限开启后自动返回并更新状态

### 3. 主界面（Main Window）
- **作用**：应用的主控制界面，用于管理悬浮框
- **显示内容**：
  - 悬浮框开关按钮
  - 悬浮框参数设置（初始大小、颜色、透明度）
  - 当前悬浮框状态显示
  - 退出应用按钮
- **交互逻辑**：
  - 点击开关按钮启动/停止悬浮框服务
  - 修改参数后实时预览效果

### 4. 悬浮框（Overlay Widget）
- **作用**：核心功能组件，用于遮挡视频字幕
- **显示内容**：
  - 半透明矩形框（带细边框）
  - 拖动控制点（四角和四边中点）
  - 最小化按钮（箭头图标）
- **交互逻辑**：
  - 单指拖动改变位置
  - 拖动控制点调整大小
  - 双击切换颜色
  - 长按弹出快捷菜单
  - 点击最小化按钮缩成小圆点

### 5. 快捷菜单（Quick Menu）
- **作用**：提供悬浮框的快捷操作
- **显示内容**：
  - 透明度调节滑块
  - 颜色选择按钮
  - 最小化按钮
  - 关闭按钮
- **交互逻辑**：
  - 长按悬浮框时弹出
  - 点击菜单外区域关闭菜单

### 6. 设置页面（Settings Page）
- **作用**：详细配置应用参数
- **显示内容**：
  - 悬浮框默认大小设置
  - 默认颜色和透明度设置
  - 记忆功能开关
  - 关于应用信息
- **交互逻辑**：
  - 修改设置后自动保存
  - 返回时应用新设置

## 技术框架

### 1. 开发语言与框架
| 技术 | 选择 | 说明 |
|-----|------|------|
| 主语言 | Python | 项目要求使用Python开发 |
| UI框架 | Kivy | 跨平台UI框架，支持Android，提供丰富的手势处理API |
| 打包工具 | Buildozer | Kivy官方提供的Android打包工具，可将Python应用打包为APK |

### 2. 核心技术方案

#### 2.1 悬浮窗实现
```python
# 技术原理
1. 使用Kivy的Window API创建悬浮窗
2. 请求Android的SYSTEM_ALERT_WINDOW权限
3. 使用Android Service在后台保持悬浮框运行
4. 设置Window FLAG_NOT_TOUCHABLE实现点击穿透
```

#### 2.2 手势处理
```python
# 技术原理
1. 使用Kivy的Touch事件系统处理手势
2. 实现拖动：记录触摸起点，计算位移量，更新悬浮框位置
3. 实现缩放：检测多点触控，计算两点间距离变化，调整悬浮框大小
4. 实现双击：检测快速连续两次触摸事件
5. 实现长按：检测触摸持续时间超过阈值
```

#### 2.3 权限管理
```python
# 技术原理
1. 使用Android的Settings.canDrawOverlays()检测权限状态
2. 使用Intent跳转到系统悬浮窗权限设置页面
3. 使用onActivityResult()获取权限设置结果
```

#### 2.4 数据持久化
```python
# 技术原理
1. 使用Android的SharedPreferences存储配置数据
2. 通过Kivy的Android插件访问SharedPreferences
3. 存储数据包括：悬浮框位置、大小、颜色、透明度等
```

## UI设计规范

### 1. 配色方案
| 颜色模式 | 颜色值 | 透明度 | 说明 |
|---------|--------|--------|------|
| 半透明灰色（默认） | #808080 | 50% | 默认模式，兼顾遮挡效果和视觉舒适度 |
| 半透明黑色 | #000000 | 50% | 增强遮挡效果，适合亮色字幕 |
| 纯黑色不透明 | #000000 | 100% | 完全遮挡，适合特殊场景 |

### 2. 布局规范
- **悬浮框**：矩形，圆角半径4px，边框宽度2px
- **主界面**：卡片式布局，每个功能模块一个卡片
- **按钮**：圆角矩形，高度48dp，内边距16dp
- **间距**：组件间距16dp，内边距12dp

### 3. 字体字号规范
| 元素 | 字号 | 字重 | 说明 |
|-----|------|------|------|
| 标题 | 24sp | Bold | 页面标题 |
| 副标题 | 18sp | Medium | 卡片标题 |
| 正文 | 14sp | Regular | 说明文字 |
| 按钮文字 | 16sp | Medium | 按钮文字 |

### 4. 响应式设计
- **屏幕适配**：使用dp和sp单位，适配不同屏幕密度
- **悬浮框尺寸**：按屏幕宽度和高度的百分比计算
- **布局适配**：使用Kivy的BoxLayout和GridLayout实现自适应布局

### 5. 交互设计
- **拖动反馈**：拖动时悬浮框略微放大（1.05倍），释放时恢复
- **边界反馈**：拖动到屏幕边缘时显示边界提示线
- **点击反馈**：按钮点击时颜色加深，显示涟漪效果
- **长按反馈**：长按时悬浮框轻微震动（如果设备支持）
